# Supabase — КНОПКА

## Синхронизация кабинета (`knopka_workspace`)

Чтобы данные кабинета (фактура, стратегия, «Данные») сохранялись в облаке и подтягивались на другом устройстве:

1. Открой [Supabase Dashboard](https://supabase.com/dashboard) → свой проект → **SQL Editor** → **New query**.
2. Скопируй содержимое файла `migrations/001_knopka_workspace.sql` и нажми **Run**.

После этого клиент в приложении (`WorkspaceSync`) сможет читать и писать строку `knopka_workspace` для текущего пользователя (RLS: только своя строка).

Если SQL не выполнен, в консоли браузера будет предупреждение — приложение продолжит работать с локальным `localStorage`.

## История чата GigaChat (`gigachat_strategy_thread`)

Чтобы диалог на странице **«Стратегия»** сохранялся в облаке для каждого пользователя (отдельная история, RLS):

1. SQL Editor → выполни **`migrations/002_gigachat_strategy_thread.sql`**.

Без миграции чат по-прежнему работает, но история только в памяти вкладки. Лимит на сервере: в коде хранится хвост из последних **120** сообщений на пользователя — нормально для Free tier и масштаба до десятков тысяч пользователей при умеренной длине ответов.
